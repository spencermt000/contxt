# Connection to V-vector norms and the layer redundancy work
This ties back to the earlier KV cache analysis in an interesting way. The V-vector norm at each layer for each token position is a measure of "how much information this position contributes to attention outputs downstream." High-V-norm positions are the ones other tokens lean on.
Hypothesis worth testing: positions with high Jacobi disagreement and high V-norm in late layers are the highest-priority correction targets. The model's belief is wrong there, and downstream tokens depend on that position. Fixing it has cascading positive effects. Conversely, high-disagreement-but-low-V-norm positions are isolated errors — worth fixing for that specific output but unlikely to generalize.

# Teacher-forced logit ranks — run the model on the correct answer with teacher forcing, look at the rank of each correct token in the predicted distribution. This is what the nudge tier classifier already uses (rank 2-10 → logit nudge, rank 10-100 → layer-local, etc.). It tells you "the model assigned probability 0.03 to the correct token here." But it doesn't tell you what the model wanted to say instead, and it conditions every position on the correct prefix — which hides cascading errors. If the model would have gotten token 5 wrong given its own (also wrong) token 4, teacher forcing never reveals that because token 4 was force-corrected.
Free-running rollouts — let the model generate freely and compare to the correct answer. This reveals cascading errors but doesn't isolate them. Once the model diverges at token 4, everything from token 4 onward is generated from a wrong prefix, so disagreements at positions 5, 6, 7 might be entirely caused by the position-4 error rather than independent problems.
Jacobi with correct initialization — gives you the best of both. Every position sees the correct context (like teacher forcing) but you observe what the model would change if allowed to (like free running). Disagreements localize to the actual problem tokens rather than cascading.

The problem we're actually trying to solve
In standard RLHF with PPO, credit assignment is handled by the value function and GAE — you propagate the scalar reward back through the trajectory, and the value head learns which states/actions contributed. In DPO you sidestep it entirely by learning from preference pairs at the sequence level. In GRPO, advantages come from group-relative scoring across multiple rollouts, also basically sequence-level.
None of these tell you which specific tokens in a bad response were the problem. They treat the trajectory as a unit. For a small local model where you want to do surgical updates (the nudge system), that's exactly the wrong granularity. You need per-token credit assignment, and you need it cheap — no value network, no preference pairs, no group rollouts.
That's the gap Jacobi decoding can fill, if used cleverly.
How Jacobi decoding gives you credit assignment for free
Standard autoregressive decoding generates one token at a time, each conditioned on all prior tokens. Jacobi decoding generates all tokens in parallel, treating it as a fixed-point iteration: initialize a guess for the full sequence, run the model on it, get the model's preferred next-token at every position, replace the guess with those predictions, repeat until convergence. Greedy autoregressive is a fixed point of this iteration — the sequence the model would have produced one token at a time is also the sequence where, given that whole sequence as input, the model's per-position predictions match the sequence itself.
The key insight for credit assignment: you can initialize Jacobi with any sequence you want, not just a model guess. Initialize it with the correct answer tokens. Run one iteration. The positions where the model's prediction agrees with the correct token are positions the model is comfortable with — given the correct context, it would output the correct token. The positions where it disagrees — where it wants to change the token to something else — are positions where the model's internal beliefs diverge from the correct answer. Those are the tokens the model "doesn't believe in" even when shown the right answer.
